# Perplexity 2.0

A full-stack AI conversational search assistant that streams responses in real-time while transparently showing its web search process. Built with a **Next.js** frontend and a **Python FastAPI + LangGraph** backend.

---

## Features

- **Streaming AI Responses**: Token-by-token streaming via Server-Sent Events (SSE) for a snappy, interactive feel.
- **Transparent Web Search**: Visually shows when the AI is _searching the web_, _reading sources_, and _writing the answer_.
- **Conversational Memory**: Multi-turn chat with persistent thread state via LangGraph's in-memory checkpointing.
- **Markdown Rendering**: AI responses support rich formatting (headings, lists, code blocks, etc.) via GitHub-Flavored Markdown.
- **Modern UI**: Clean, responsive chat interface built with Tailwind CSS.

## Getting Started

### 1. Clone the repository

```bash
git clone <repo-url>
cd <repo-directory>
```

### Create a virtual environment (recommended)

python -m venv venv
source venv/bin/activate # On Windows: venv\Scripts\activate

## 2. Server Setup

pip install -r requirements.txt

### Add your API keys to env file
GOOGLE_API_KEY="your-google-api-key"
TAVILY_API_KEY="your-tavily-api-key"

### Run the FastAPI server
uvicorn server.app:app --reload

## 3. Client Setup

cd client
npm install
npm run dev

# How It Works
Agent Architecture (LangGraph)
The backend defines a simple but powerful state graph:

model — The Gemini LLM receives the conversation history and decides whether to respond directly or call a tool.
tool_node — If the model requests a tavily_search_results_json, the tool executes the search and returns results back to the model.
Conditional Edge — After the model node, the graph routes to tool_node if tool calls exist, otherwise ends the turn.
Conversation state is persisted using LangGraph's MemorySaver with a unique thread_id, enabling multi-turn chat across separate HTTP requests.

Streaming Protocol
The client opens a POST /chat_stream SSE connection. The server emits typed JSON events:

| Event Type     | Purpose                                                                  |
| -------------- | ------------------------------------------------------------------------ |
| checkpoint     | Returns a checkpoint_id for new conversations to preserve thread memory. |
| content        | Token-by-token streaming of the AI's text response.                      |
| search_start   | Signals the AI is about to search the web; includes the query.           |
| search_results | Returns the list of source URLs retrieved from Tavily.                   |
| end            | Signals the response stream is complete.                                 |

Search Status UI
When the AI triggers a search, the frontend renders a live stepper above the assistant's message:

Searching the web — displays the query generated by the LLM
Reading — displays the fetched source URLs
Writing answer — final stage while the model synthesizes the response
